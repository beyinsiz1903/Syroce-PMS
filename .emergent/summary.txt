**Key Technical Concepts**
- **Multi-Provider Architecture:** A framework supporting both REST and SOAP APIs through common interfaces.
- **Idempotent & Versioned Ingest:** A resilient data pipeline using  and payload hashing to prevent stale or duplicate data.
- **Event-Driven ARI Push:** An asynchronous architecture where inventory/rate changes are processed via a buffered, coalescing pipeline to prevent API spam and ensure efficient updates.
- **Debouncing & Coalescing:** Merging high-volume, similar events in a short time window into a single, compact change set.
- **Delta Compilation:** Generating the minimum possible payload to send to a provider (e.g., one date-range update instead of seven daily ones).
- **Provider Adapter Pattern:** Translating a canonical internal data model () into the specific format required by each external provider.
- **Drift Worker:** A background process that periodically compares PMS and provider data to detect and correct discrepancies.
