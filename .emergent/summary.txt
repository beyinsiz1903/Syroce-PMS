**Key Technical Concepts**
-   **Strangler Fig Pattern:** Incrementally replacing legacy functionality by wrapping it with new services and redirecting calls (bridging).
-   **Semantic Core / Domain-Driven Design:** Organizing code around business domains (, , , ) and their interactions, rather than technical layers.
-   **Outbox Pattern:** Ensuring reliable event emission by writing events to a dedicated  table within the same transaction as the business operation.
-   **Idempotency:** Preventing duplicate operations for write requests by tracking an  provided by the client.
-   **Shadow Comparison:** Running new and old code paths in parallel for read operations and logging any differences to ensure the new implementation is correct before a full cutover.
-   **Test Hardening:** Using contract snapshot tests and explicit tenant/property isolation tests to prevent regressions during migration.
