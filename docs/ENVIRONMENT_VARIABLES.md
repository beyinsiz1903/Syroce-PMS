, `REVENUE_AUTOPILOT_EMISSION_ENABLED` (RevenueAutopilot canliya-gecis GLOBAL kill-switch; fail-closed default `false`; yalnizca acikca `true` ise emisyon mumkun), `REVENUE_AUTOPILOT_PILOT_TENANTS` (virgulle ayrilmis PILOT tenant_id allowlist'i; bos=hicbir tenant). **Cift kilit (fail-closed):** ikisi birden saglanmadan tenant DB'de `full_auto` yazili olsa bile RATE_UPDATED emisyonu + kanal push YAPILMAZ; sistem shadow (supervised) moduna duser ve durum gorunur olur (`revenue_autopilot_runs.last_emission_gated`, status API `emission_gated`/`effective_mode`) — sessiz fallback DEGIL.
, `FEEDBACK_UNIFIED_READ_ENABLED` (NPS raporlama fiziksel cutover okuma anahtari; fail-closed default `false`). Default (`false`) iken canli `/nps/*` uclari AUTHORITATIVE `nps_surveys`'ten okur (sifir-degisim). `true` iken `compute_nps_score`/`recent_feedback`/`by_room` kanonik `feedback_entries` koleksiyonundan `nps_eligible=True` filtresiyle okur; cikti legacy ile birebir kalir (1-5 yildiz yorum + anket yanitlari NPS matematigine KARISMAZ). **Cutover sirasi:** (1) yazimlar zaten kanonik modele dual-write yapar (her zaman acik, en-iyi-caba, idempotent dedup_key), (2) `python -m scripts.migrate_feedback_unified --apply` ile backfill, (3) `python -m scripts.migrate_feedback_unified --verify` parity YESIL olmali (legacy vs kanonik NPS skoru birebir), (4) ancak o zaman bu flag `true` yapilir. Parity kirmizi iken flag ACILMAMALI.
