## Doctrine compliance (F8L 50/51/52 mirror)

- **`stress_tid` scope** — tüm write'lar stres tenant'ta; bulk-resolve
  payload'ı `stress_token` ile gönderiliyor; pilot_token yalnız read-only
  baseline + IDOR negative control için.
- **`pilot_drift=0`** — Z-step her run sonunda enforce; ek olarak F-step
  (cross-tenant IDOR) pre/post pilot booking room_id snapshot karşılaştırması.
- **`external_calls=[]`** — Z-step `assertNoExternalCallsPostBatch`; CB
  read pure-mem, bulk-resolve `room_night_locks` atomic insert
  (outbound HTTP YOK).
- **Module-blocked pattern** — CB probe veya CQ probe non-2xx →
  ilgili alt-grup test'leri `test.skip()` + P2 informational; Z-step
  bağımsız koşar.
- **failedTests=0, P0=P1=0** — error-path coverage deterministic
  sentetik ID'lerle assert edilir; gerçek pending booking eksikse
  P2 REVIEW (G test'i informational, fake-PASS YOK).

## Pending-seed gap (P2 informational, kabul edildi)

Stres seed pipeline (`backend/domains/admin/router/stress.py`)
pending_assignment booking üretmez (`allocation_source="manual"`).
Gerçek pending bookings yalnız OTA import path'inden
(`backend/channel_manager/application/reservation_import_service.py`
+ `core.atomic_booking.BookingConflictError` fallback) çıkar — bu da
external_calls invariant'ını ihlal eder (gerçek HR/Exely çağrısı
gerektirir). Bu yüzden:

- **Error-path coverage (B/C/D/E/F):** deterministic, sentetik ID'lerle
  tam kapsam (room_not_found, not_pending, dedup, max 50, anon, IDOR).
- **Real-succeeded coverage (G):** yalnız setup'ta existing pending
  bulunursa çalışır; yoksa P2 REVIEW informational. Partial-failure
  ISOLATION sözleşmesi B test'iyle zaten kanıtlanır (2 item, 2 bağımsız
  sonuç, top-level error YOK).

Future work: backend stress seed'e opsiyonel `seed_pending_bookings:N`
parametresi eklenirse G test'i real coverage'a yükseltilir (out-of-scope
bu task'ta — backend hardening değişikliği değil).
