Syroce PMS Mobil Uygulama — Geliştirme Promptu
Hedef
Mevcut Syroce PMS web uygulamasının (FastAPI backend + MongoDB Atlas) resmi mobil istemcisini geliştir. iOS ve Android'de native hızında çalışan, otel personelinin sahada (resepsiyon, kat hizmetleri, müdür) ve misafirin cebinde (guest app) tek elle, üç saniyede iş bitirebileceği bir uygulama olacak.

Teknik Yığın
Framework: Expo (React Native) + TypeScript + expo-router
State: TanStack Query (server state) + Zustand (UI state)
UI: Tamagui veya NativeWind (Tailwind), karanlık mod zorunlu
Auth: mevcut backend /api/auth/login JWT, expo-secure-store'da sakla
Push: expo-notifications + backend'e device token kaydı
Offline: TanStack Query persistor + MMKV; kritik listeler (in-house, bugünkü check-in/out) offline okunabilir
API Base: environment ile değişen EXPO_PUBLIC_API_URL (backend 8000, prod ayrı)
Dil: Türkçe (i18n hazır), jargonsuz, emoji yok
4 Rol, 4 Ana Akış
Aynı binary, login sonrası rolüne göre farklı tab seti:

1. Resepsiyon (front_desk)
Bugün sekmesi: girişler, çıkışlar, walk-in butonu, no-show riski yüksek rezervasyonlar
Hızlı Check-in: QR/barkod tara → kimlik fotoğrafı çek (Quick-ID 8099 servisine yolla) → oda seç → tamam
Hızlı Check-out: oda no yaz → folio göster → ödeme al → fatura paylaş
Walk-in 30 saniye: ad + telefon + oda tipi → uygun ilk oda → fiyat onay → check-in
Misafir ara: ad/oda/telefon → profil (VIP rozeti, kara liste uyarısı, tercihler)
2. Kat Hizmetleri (housekeeping)
Oda listesi: kat filtresi, durum renkleri (kirli/temiz/inceleme/arıza)
Tek dokunuş durum değiştir: uzun bas → temizleniyor/temiz/inceleme bekliyor
Hasar bildir: fotoğraf çek + not + öncelik → backend'e push
Acele temizlik: erken giriş talebi olan odalar üstte, kırmızı rozet
Kayıp eşya: fotoğraf + oda + tarih → kayıt
3. Müdür (manager / gm)
Dashboard: doluluk, ADR, RevPAR, bugünkü gelir, açık şikayet, düşük puan değerlendirme — tek ekran
VIP geliyor kartı: fotoğraf + tercih notu + oda + saat
Anlık alarmlar: overbooking riski, düşük stok, kanal kopması
Onay kuyruğu: indirim talepleri, oda upgrade, iade — tek dokunuş onay/red
Vardiya devir notu oku/yaz
4. Misafir (guest_app)
Rezervasyonlarım: aktif + geçmiş, otel/oda/tarih kartları
Online check-in: check-in günü 06:00'dan itibaren açık, kimlik fotoğrafı + imza
Dijital anahtar: check-in sonrası QR (resepsiyon kapısı + oda kapısı)
Oda servisi: menü + sepet + sipariş takip
Mesajlaşma: otel ile chat (read receipt)
Sadakat: puan, seviye, faydalar — çok otelli (tüm Syroce property'lerinden toplam)
Erken giriş / geç çıkış talep: fiyat backend kuralı ile anında hesaplanıp gösterilir
Backend Entegrasyon Kuralları
Sıfır yeni endpoint öncelik: mevcut REST API'yi kullan (/api/bookings, /api/rooms, /api/guests, /api/folios, /api/notifications/mobile/*, /api/walkin, /api/shift-handover, /api/eod-report, /api/early-late-pricing, /api/no-show-today, /api/blacklist, /api/room-map, /api/guest/*)
Eksik mobil-özel uç çıkarsa (örn. push token kaydı) backend/domains/pms/mobile_router/ altına ekle, mevcut notifications.py paterniyle
Tenant izolasyonu: her istekte JWT'deki tenant_id zorunlu, hiçbir istemci tarafı override yok
Quick-ID: kimlik OCR için 8099 portuna multipart POST, dönen JSON'u backend'in /api/guests create'ine maple
UX Standartları (zorunlu)
Tek el kullanım: tüm aksiyon butonları ekranın alt 1/3'ünde (FAB veya bottom sheet)
Üç saniye kuralı: ana akış (check-in/check-out/temizlik) en fazla üç dokunuş
Haptic feedback: her başarı/hata için
Skeleton loader: spinner yerine
Pull-to-refresh: her listede
Offline rozeti: üstte ince bant "çevrimdışı, son güncelleme 2 dk önce"
Hata mesajları: teknik kod yok, "Bağlantı kurulamadı, tekrar dene" gibi
Erişilebilirlik: dynamic font, VoiceOver/TalkBack etiketleri, kontrast AA
Güvenlik
JWT 15 dk + refresh token, biyometrik (Face ID / parmak izi) ile uygulama kilidi
Kimlik fotoğrafları cihazda kalmaz, upload sonrası silinir
Personel cihazından çıkış yapılınca expo-secure-store temizlenir
